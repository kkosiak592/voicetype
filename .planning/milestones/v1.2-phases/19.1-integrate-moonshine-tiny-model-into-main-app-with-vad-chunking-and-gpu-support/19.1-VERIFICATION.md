## Human Verification Required

### 1. Short-audio transcription correctness

**Test:** With Moonshine selected as the active engine, press and hold the hotkey, say a short phrase (~5 seconds), release.
**Expected:** The phrase appears correctly at the cursor within approximately 500ms.
**Why human:** Inference correctness (not just that inference runs) cannot be verified from code alone.

### 2. Long-audio VAD chunking correctness (>30 seconds)

**Test:** With Moonshine selected, record a continuous dictation lasting at least 35 seconds.
**Expected:** The full phrase appears correctly — no garbled, repeated, or truncated text at chunk boundaries.
**Why human:** VAD chunking boundary quality depends on audio signal characteristics; static code review cannot confirm the stitched output is coherent.

### 3. Bidirectional engine switching

**Test:** Start with Whisper active. Switch to Moonshine in Settings > Model. Dictate. Switch back to Whisper. Dictate again.
**Expected:** Both switches succeed (no error toasts, Inference Status updates), and both engines produce correct text.
**Why human:** Live model loading/unloading and state transition correctness requires running app.

### 4. Persistence across restart

**Test:** Set Moonshine as the active engine. Close the app fully. Relaunch.
**Expected:** Moonshine is still selected after relaunch — no fallback to Whisper, startup loading log confirms "Moonshine model loaded at startup".
**Why human:** Startup loading path (lib.rs:1849-1899) requires an actual restart to exercise.

### 5. GPU execution provider (NVIDIA machine only)

**Test:** On a machine with an NVIDIA GPU and CUDA, select Moonshine and check Inference Status in Settings.
**Expected:** Provider field shows "CUDA". First inference after loading is slower (CUDA context init warm-up), subsequent ones are fast (~191ms for 5s clips per research).
**Why human:** CUDA EP selection requires NVIDIA hardware and cannot be inferred from code review alone.
