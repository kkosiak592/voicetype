**Double-click tray handler opens settings window; destroy_tray IPC hides icon before relaunch to prevent duplicate icons on Windows**

## Accomplishments

- Added `TrayIconEvent::DoubleClick` handler with `MouseButton::Left` guard to `build_tray()` — double-clicking the tray icon now shows and focuses the settings window
- Added `destroy_tray()` public function to tray.rs using `set_visible(false)` to remove the tray icon before relaunch
- Added `destroy_tray` IPC command in lib.rs, registered in `invoke_handler`
- Updated updater.ts `restartNow()` to call `invoke('destroy_tray')` in try/catch before `relaunch()` — prevents duplicate tray icons after auto-update restart on Windows

## Files Created/Modified

- `src-tauri/src/tray.rs` - Added `MouseButton` and `TrayIconEvent` imports; chained `.on_tray_icon_event()` handler on TrayIconBuilder; added `pub fn destroy_tray()`
- `src-tauri/src/lib.rs` - Added `#[tauri::command] fn destroy_tray()` and registered it in `invoke_handler`
- `src/lib/updater.ts` - Added `invoke('destroy_tray')` with try/catch before `await relaunch()` in `restartNow()`

## Decisions Made

- `set_visible(false)` used for `destroy_tray` — Tauri v2 `TrayIcon` type does not expose a public `destroy()` method; setting visibility to false removes the icon from the Windows system tray
- `MouseButton::Left` guard on DoubleClick handler — avoids right-double-click (which can happen on some mice) unintentionally opening settings
- try/catch around `invoke('destroy_tray')` in updater.ts — non-critical cleanup; relaunch must proceed even if tray already gone or IPC unavailable
