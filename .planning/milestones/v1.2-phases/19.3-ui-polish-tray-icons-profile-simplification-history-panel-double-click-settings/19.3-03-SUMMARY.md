# Phase 19.3 Plan 03: Transcription History Panel Summary

**One-liner:** Persistent history panel with JSON storage, pipeline recording, and click-to-copy React UI.

## What Was Built

Backend: `history.rs` module with `HistoryEntry` struct (text, timestampMs, engine), `HistoryState` mutex-backed managed state, `load_history`/`save_history`/`append_history` helpers persisting to `history.json` in the app data dir. History capped at 50 entries, newest first.

`lib.rs`: `mod history` declaration, `get_history` IPC command, `HistoryState` loaded from disk in `setup()` before `build_tray`, registered in `invoke_handler`.

`pipeline.rs`: After successful injection, reads `ActiveEngine` to get engine name string and calls `history::append_history(&app, &formatted_for_tooltip, engine_name)`.

Frontend: `HistorySection.tsx` fetches history on mount via `invoke('get_history')`, renders scrollable list of entries with text (100-char truncation, full text on hover), formatted timestamp, engine badge, and click-to-copy with 1.5s "Copied!" feedback. Empty state message shown when no entries. `App.tsx` replaces history placeholder div with `<HistorySection />`.
