<task type="auto">
  <name>Task 1: Create GitHub Actions release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create `.github/workflows/` directory and the `release.yml` workflow file with the following structure:

**Trigger:** `on: push: tags: ['v*']` — fires on any tag starting with v (per CICD-01).

**Job: `publish-release`**
- `runs-on: windows-latest`
- `permissions: contents: write` (needed for release creation)

**Steps (in order):**

1. **Checkout** — `actions/checkout@v4`

2. **Setup Node.js** — `actions/setup-node@v4` with `node-version: lts/*`

3. **Install Rust stable** — `dtolnay/rust-toolchain@stable`

4. **Rust cache** — `Swatinem/rust-cache@v2` with `workspaces: src-tauri` and `cache-on-failure: true`. This caches the cargo registry and target dir to speed up subsequent builds. The first build will still be slow (20-40 min) but incremental builds will benefit.

5. **Install CUDA Toolkit** — Use `Jimver/cuda-toolkit@v0.2.21` (well-maintained action) with:
   - `cuda: '12.6.3'` (latest stable 12.x — whisper-rs + parakeet-rs need CUDA headers + nvcc)
   - `method: 'network'`
   - `sub-packages: '["nvcc", "cudart", "cublas", "cublas_dev", "thrust", "visual_studio_integration"]'` — minimal subset for compilation, NOT the full 4GB toolkit
   After this step, CUDA_PATH is automatically set by the action.

6. **Install LLVM/clang** — Use `KyleMayes/install-llvm-action@v2` with:
   - `version: '18'`
   - `directory: ${{ runner.temp }}/llvm`
   This sets LIBCLANG_PATH automatically. Required by whisper-rs bindgen.

7. **Set build environment variables** — Shell step that sets:
   ```
   echo "CMAKE_CUDA_ARCHITECTURES=61;75;86;89" >> $GITHUB_ENV
   ```
   Multi-arch so the binary supports Pascal (61), Turing (75), Ampere (86), and Ada Lovelace (89) GPUs. CMAKE_CUDA_ARCHITECTURES is read by whisper.cpp's cmake build.

8. **Install frontend dependencies** — `run: npm ci` (use `ci` not `install` for reproducible CI builds)

9. **Build and release with tauri-action** — `tauri-apps/tauri-action@v0` with:
   ```yaml
   env:
     GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
     TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
     TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
   with:
     tagName: ${{ github.ref_name }}
     releaseName: 'VoiceType ${{ github.ref_name }}'
     releaseBody: 'See the assets to download and install VoiceType ${{ github.ref_name }}.'
     releaseDraft: false
     prerelease: false
     includeUpdaterJson: true
     updaterJsonPreferNsis: true
   ```

   Key points:
   - `tagName: ${{ github.ref_name }}` — uses the pushed tag directly (e.g., `v1.0.0`) instead of `v__VERSION__` pattern, since the tag is the source of truth
   - `releaseDraft: false` — per user decision, publish immediately
   - `includeUpdaterJson: true` — generates latest.json with download URLs and Ed25519 signatures (CICD-04)
   - `updaterJsonPreferNsis: true` — ensures latest.json points to the NSIS .exe installer (not .msi) since tauri.conf.json targets NSIS
   - TAURI_SIGNING_PRIVATE_KEY env var triggers Tauri's signing during build — produces .sig files alongside installers (CICD-03)
   - tauri-action automatically creates the GitHub Release and uploads all artifacts: .exe installer, .exe.sig signature, latest.json (CICD-05)

**Do NOT include:**
- Matrix strategy (Windows-only, per user decisions)
- Test/lint steps (per user decision: build and release only)
- Draft review step (per user decision: publish immediately)
- Manual workflow_dispatch trigger (tag push is sole trigger)

**YAML formatting:**
- Use consistent 2-space indentation
- Add brief comments on non-obvious steps (CUDA sub-packages, CMAKE_CUDA_ARCHITECTURES values)
- Name every step clearly
