<task type="auto">
  <name>Task 1: Configure NSIS bundle and build installer</name>
  <files>src-tauri/tauri.conf.json</files>
  <action>
1. Update `src-tauri/tauri.conf.json` bundle section:
   ```json
   "bundle": {
       "active": true,
       "targets": ["nsis"],
       "icon": [
           "icons/32x32.png",
           "icons/128x128.png",
           "icons/128x128@2x.png",
           "icons/icon.icns",
           "icons/icon.ico"
       ],
       "windows": {
           "nsis": {
               "installMode": "currentUser",
               "displayLanguageSelector": false,
               "startMenuFolder": "VoiceType"
           }
       }
   }
   ```

   Key points:
   - `targets: ["nsis"]` — NSIS only, not "all" (avoids WiX/MSI which requires WiX toolset)
   - `installMode: "currentUser"` — installs to AppData, no UAC prompt required
   - `displayLanguageSelector: false` — English-only app per out-of-scope in REQUIREMENTS.md
   - `startMenuFolder: "VoiceType"` — clean Start Menu grouping
   - No `resources` section — models are NEVER bundled (per locked decision)

2. Verify no model files are accidentally included in bundle:
   - Confirm no `bundle.resources` entry exists
   - Confirm `src-tauri/icons/` directory does not contain model files

3. Build the NSIS installer:
   ```bash
   cd src-tauri && cargo tauri build --bundles nsis
   ```
   This will compile the Rust backend in release mode and produce the NSIS installer.
   Build time: expected 5-15 minutes (release mode with whisper CUDA compilation).

   Output: `src-tauri/target/release/bundle/nsis/VoiceType_0.1.0_x64-setup.exe`

4. Verify installer size:
   ```bash
   ls -la src-tauri/target/release/bundle/nsis/VoiceType_0.1.0_x64-setup.exe
   ```
   Must be under 5 MB. If over 5 MB, the binary likely includes debug symbols or model files — investigate.

   NOTE: The 5 MB limit may be challenging because the whisper-rs CUDA binary links against CUDA libraries. If the binary itself exceeds 5 MB (CUDA runtime linkage), document the actual size and note that the installer size constraint in DIST-03 may need revision. The intent is "models excluded" — the binary + WebView2 bootstrapper size is a build artifact of CUDA compilation.

5. Run Windows Defender scan on the produced installer:
   ```powershell
   # From PowerShell (not bash)
   & "C:\Program Files\Windows Defender\MpCmdRun.exe" -Scan -ScanType 3 -File "src-tauri\target\release\bundle\nsis\VoiceType_0.1.0_x64-setup.exe"
   ```
   Expected result: no threats detected. SmartScreen warnings are separate and expected for unsigned binaries — not a Defender issue (per RESEARCH.md pitfall 5).
