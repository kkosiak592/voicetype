<objective>
Refactor AudioCapture and WhisperState into Mutex-wrapped managed state so microphone and model can be switched at runtime without app restart.

Purpose: Users need to select their preferred microphone and whisper model from the settings panel. Both require replacing Tauri managed state at runtime — AudioCapture for mic switching (restarts audio stream) and WhisperContext for model switching (reloads whisper model). This is the primary refactoring risk in Phase 6.

Output: `AudioCaptureMutex` wrapper in audio.rs, `WhisperStateMutex` wrapper in lib.rs, device enumeration command, mic/model switching commands, persistence of selections.
</objective>

       ```rust
       #[tauri::command]
       fn list_input_devices() -> Result<Vec<String>, String> {
           use cpal::traits::HostTrait;
           let host = cpal::default_host();
           let devices = host.input_devices().map_err(|e| e.to_string())?;
           let mut names = vec!["System Default".to_string()];
           for device in devices {
               if let Ok(desc) = device.description() {
                   names.push(desc.name().to_string());
               }
           }
           Ok(names)
       }
       ```
