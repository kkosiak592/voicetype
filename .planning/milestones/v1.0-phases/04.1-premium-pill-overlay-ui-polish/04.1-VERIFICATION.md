**Note on PLAN 02 `animationend` key link:** PLAN 02 specified gating `appWindow.hide()` via `animationend` event listener on the container. The implementation uses `setTimeout(180ms)` instead — matching the exit animation duration. The outcome is equivalent: hide is always deferred until exit animation completes. This is an acceptable implementation deviation; `animationend` can be unreliable in WebView2 contexts.

**Note on `animState` implementation:** PLAN 02 said "ref or state"; SUMMARY 02 called it "ref". Actual code uses `useState<AnimState>`. This is correct — `animState` must trigger re-renders to apply CSS class changes to the container div, so `useState` is the right choice. No gap.

### Human Verification Required

All automated checks pass. The following items require visual verification because they depend on runtime animation behavior, perceptual quality, and Win32 focus behavior:

**1. Spring Entrance Overshoot**
**Test:** Launch the app, hold the hotkey, observe the pill appearing.
**Expected:** Pill expands from a tiny dot with a visible overshoot (briefly larger than final size) before settling. The `cubic-bezier(0.34, 1.56, 0.64, 1)` easing should produce a premium spring feel, not a linear pop-in.
**Why human:** Perceptual quality of spring animation cannot be verified by static analysis of CSS values.

**2. Recording-to-Processing Cross-Fade**
**Test:** Hold the hotkey, speak briefly, release the hotkey; watch the transition between recording and processing states.
**Expected:** Frequency bars fade out as three bouncing dots fade in — no visible flash, no blank frame between states. Indigo glow pulse becomes visible around the pill in processing state.
**Why human:** "No content flash" is a temporal visual property imperceptible to static code inspection.

**3. Success Checkmark Draw-On and Auto-Dismiss**
**Test:** Complete a transcription, observe the success state.
**Expected:** A checkmark draws itself from left to right over ~280ms, holds for ~320ms, then the pill scales down to a dot and vanishes. The draw animation should look smooth, not choppy.
**Why human:** SVG `stroke-dashoffset` animation smoothness and timing feel require visual inspection.

**4. Silent Error Dismiss**
**Test:** Trigger an error (empty recording, or a recording short enough to produce no transcription result).
**Expected:** Pill scales down and disappears with no text, no color change, no "No speech" message.
**Why human:** Absence of unwanted visual elements requires runtime observation.

**5. No Focus Steal**
**Test:** Open Notepad, place cursor, hold the hotkey and dictate while Notepad is the active window. Release and wait for transcription.
**Expected:** Transcribed text lands in Notepad. The pill appearing and dismissing does not redirect keyboard focus away from Notepad.
**Why human:** Win32 `WS_EX_NOACTIVATE` focus prevention is a runtime behavior verified by typing, not by reading code.
