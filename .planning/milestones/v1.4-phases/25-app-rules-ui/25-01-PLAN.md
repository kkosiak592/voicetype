  <name>Task 2: Implement detect-app flow with inline countdown on button</name>
  <files>src/components/sections/AppRulesSection.tsx</files>
  <action>
Add the detect flow to AppRulesSection.tsx (in the same file created in Task 1).

**Detect button** placed above the rules list card (or inside the card header area), styled as a primary action button:
- Emerald background (`bg-emerald-600 hover:bg-emerald-700 text-white rounded-xl px-4 py-2 text-sm font-medium`)
- Lucide `Crosshair` or `Focus` icon + "Detect Active App" label

**Detect state machine** using `useState`:
- `detectState: 'idle' | 'countdown' | 'success' | 'error'`
- `countdown: number` (3, 2, 1)
- `detectMessage: string` (success/error message)

**Countdown flow:**
1. On button click, set `detectState = 'countdown'`, `countdown = 3`
2. Use `useEffect` watching `detectState === 'countdown'`:
   - Start `setInterval` with 1000ms, decrement countdown
   - Store interval ID in `useRef` for cleanup
   - When countdown reaches 0: clear interval, call `invoke<DetectedApp>('detect_foreground_app')`
   - On success:
     - If `exe_name` is null: set error state "Could not detect app -- try again"
     - If exe_name already in rules: set success state "already configured" (or similar -- Claude's discretion: show "{exe_name} already added" message on button)
     - Otherwise: call `invoke('set_app_rule', { exeName: exe_name, rule: { all_caps: null } })`, add to local rules state with `all_caps: null` (inherit default), store window_title in `windowTitles` if available, set success state "Added {exe_name}"
   - On failure: set error state "Could not detect app -- try again"
3. Success/error message shows on the button text itself for 2.5 seconds (per locked decision: inline on button, no toast/modal), then resets to 'idle'
4. **Cleanup:** Return cleanup function from useEffect that clears the interval (handles unmount during countdown)

**Button text by state:**
- `idle`: icon + "Detect Active App"
- `countdown`: "Detecting in {countdown}..." (no icon, or a spinner)
- `success`: checkmark icon + "{detectMessage}" in green
- `error`: "{detectMessage}" in red/amber

**Button disabled** during countdown, success, and error states (prevent double-clicks).

Per locked decision: No modal, toast, or confirmation step. Everything happens in-place on the button.
