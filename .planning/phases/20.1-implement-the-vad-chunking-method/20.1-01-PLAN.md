<objective>
Generalize the Moonshine-specific VAD chunking into an engine-agnostic function and wire it into Whisper and Parakeet engine dispatch, so all three engines handle 60s+ recordings via VAD-based chunk splitting at silence boundaries.

Purpose: Whisper has O(n^2) self-attention scaling and Parakeet's TDT decoder slows on long sequences. Chunking at silence boundaries (already proven in the Moonshine path) gives 30-50% latency reduction for 60s+ recordings with no accuracy loss.

Output: Three modified Rust files -- `vad.rs` (generic function), `pipeline.rs` (Whisper/Parakeet chunk loops), `transcribe_moonshine.rs` (refactored to use generic function).
</objective>

Modify the engine dispatch section of `run_pipeline()` in `pipeline.rs` to add VAD chunking for Whisper and Parakeet when audio exceeds 30 seconds.

**CRITICAL anti-pattern from RESEARCH.md:** The VAD chunking (`vad_chunk_audio`) must happen BEFORE `spawn_blocking`, not inside it. The VAD uses ONNX Runtime and the chunk list should be ready before entering the blocking context.

**For the Whisper arm (line ~170, `crate::TranscriptionEngine::Whisper`):**

After the `ctx` extraction (line ~187, after the `None` early return), add the chunking + chunk loop:

```rust
// VAD chunk for audio >30s — compute chunks BEFORE spawn_blocking
// (VAD uses ONNX Runtime; keep it outside the blocking context)
const CHUNK_THRESHOLD_SAMPLES: usize = 30 * 16000;
let chunks = if samples.len() > CHUNK_THRESHOLD_SAMPLES {
    log::info!(
        "Pipeline: Whisper audio {:.1}s exceeds 30s — splitting with VAD",
        samples.len() as f32 / 16000.0
    );
    vad::vad_chunk_audio(&samples, 30)
} else {
    vec![samples]
};
match tauri::async_runtime::spawn_blocking(move || {
    let mut combined = String::new();
    for chunk in &chunks {
        let text = crate::transcribe::transcribe_audio(&ctx, chunk)?;
        let trimmed = text.trim();
        if !trimmed.is_empty() {
            if !combined.is_empty() {
                combined.push(' ');
            }
            combined.push_str(trimmed);
        }
    }
    Ok::<String, String>(combined)
})
.await
```

Replace the existing `spawn_blocking` call that passes `samples` directly to `transcribe_audio`. The error handling arms (`Ok(Ok(text))`, `Ok(Err(e))`, `Err(e)`) remain unchanged.
