**Step 1: Compute SHA256 of the upstream file.**

Download `https://huggingface.co/distil-whisper/distil-large-v3.5-ggml/resolve/main/ggml-model.bin` and compute its SHA256 hash. On Windows: `certutil -hashfile ggml-model.bin SHA256`. This is a 1.52 GB download. The resulting hash is used in `model_info()` below. Save the file to the VoiceType models directory as `ggml-distil-large-v3.5.bin` (rename from the generic `ggml-model.bin`).

**IMPORTANT:** If the download is impractical in the executor environment (1.52 GB), use the HuggingFace API to get the file hash instead:
```bash
curl -s "https://huggingface.co/api/models/distil-whisper/distil-large-v3.5-ggml/revision/main" | python -c "import sys,json; files=json.load(sys.stdin)['siblings']; print([f for f in files if f['rfilename']=='ggml-model.bin'][0])"
```
The LFS metadata from HuggingFace may include a sha256 in the `lfs` field. If neither approach yields the hash, leave a `TODO` comment with the placeholder and flag this for manual resolution.

**Step 2: Handle 4-card grid layout.**

The current `gridClass` uses `lg:grid-cols-3` for 3+ models. With 4 visible models (GPU users), the grid will show 3+1 layout. This is acceptable per research. No grid change needed — the existing responsive breakpoints handle it.

However, if you want to improve the 4-card layout, you may optionally add `xl:grid-cols-4` when `visibleModels.length >= 4`:

```typescript
const gridClass =
  visibleModels.length >= 4
    ? 'grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 mb-6'
    : visibleModels.length >= 3
      ? 'grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3 mb-6'
      : visibleModels.length === 2
        ? 'grid grid-cols-1 gap-4 sm:grid-cols-2 mb-6'
        : 'grid grid-cols-1 gap-4 mb-6';
```

This is Claude's discretion — the cosmetic improvement is optional. If added, the 4 cards tile neatly on extra-large screens.
