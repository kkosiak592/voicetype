  <action>
1. Add `voice_activity_detector = "0.2.1"` to `[dependencies]` in Cargo.toml. Add a comment: `# Silero VAD V5 — neural speech/silence detection (ONNX Runtime downloaded at build time)`. Place it after the tokio entry.

2. Create `src-tauri/src/vad.rs` with:

   a. **Constants** (per CONTEXT.md locked decisions and RESEARCH.md):
      - `SPEECH_PROBABILITY_THRESHOLD: f32 = 0.5` (Silero default)
      - `SILENCE_FRAMES_THRESHOLD: u32 = 47` (~1.5s at 32ms/chunk = 1504ms)
      - `MIN_SPEECH_FRAMES: u32 = 9` (~300ms at 32ms/chunk = 288ms)
      - `MAX_RECORDING_FRAMES: u32 = 1875` (60s safety cap per Claude's discretion)
      - `CHUNK_SIZE: usize = 512` (Silero V5 fixed chunk size at 16kHz)

   b. **`vad_gate_check(samples: &[f32]) -> bool`** — synchronous post-hoc VAD gate for hold-to-talk mode:
      - Creates a fresh `VoiceActivityDetector` (sample_rate 16000, chunk_size 512)
      - Iterates over `samples.chunks(CHUNK_SIZE)`, calls `vad.predict()` on each
      - Counts frames where probability >= SPEECH_PROBABILITY_THRESHOLD
      - Returns `true` if speech_frames >= MIN_SPEECH_FRAMES (enough speech to transcribe)
      - Returns `false` if insufficient speech (caller should discard)
      - Log the speech frame count and total frames processed at info level

   c. **`VadWorkerHandle`** struct — cancel handle for the streaming VAD worker:
      - Contains `cancel_tx: Option<tokio::sync::oneshot::Sender<()>>`
      - Method `cancel(&mut self)` — takes the sender and sends cancellation signal
      - This struct will be stored in managed state as `Arc<Mutex<Option<VadWorkerHandle>>>` (Plan 02 will wire this)

   d. **`VadWorker`** — no struct needed, just a public function:
      - `pub fn spawn_vad_worker(app: tauri::AppHandle, buffer: Arc<Mutex<Vec<f32>>>) -> VadWorkerHandle`
      - Spawns a `tauri::async_runtime::spawn` async task
      - Creates a fresh `VoiceActivityDetector` per session (avoid stale LSTM state — Pitfall 4 from RESEARCH)
      - Reads buffer by cursor position: `buffer.lock()` (blocking OK — VAD is not on the audio callback thread — Pitfall 5)
      - Processes 512-sample chunks, tracks `silence_frames`, `speech_frames`, `ever_spoke`, `total_frames`
      - When `silence_frames >= SILENCE_FRAMES_THRESHOLD` and `ever_spoke`:
        - If `speech_frames >= MIN_SPEECH_FRAMES`: set auto-stop flag and trigger pipeline
        - If insufficient speech: discard (emit pill-result error, reset to idle)
      - When `total_frames >= MAX_RECORDING_FRAMES`: treat as auto-stop (60s safety cap)
      - Check `cancel_rx.try_recv()` each iteration — break on cancellation (second tap or hold-to-talk release)
      - On auto-stop trigger: transition PipelineState from RECORDING to PROCESSING via CAS, stop audio recording, stop level stream, emit pill-state processing, set tray processing, then spawn `run_pipeline()` using the full crate path (see note below). If CAS fails (already stopped by second tap), just exit silently.
      - Sleep 10ms when no new chunk available to avoid spinning
      - Use `use tauri::Manager;` and `use tauri::Emitter;` for state access and event emission

   **CRITICAL — Avoiding circular module coupling:** `pipeline.rs` imports from `vad.rs` (via `use crate::vad;` for `vad_gate_check`). To avoid circular `use` imports, `vad.rs` MUST NOT have `use crate::pipeline;` at the module top level. Instead, reference pipeline items via full crate paths inline:
   - Access PipelineState: `app.state::<crate::pipeline::PipelineState>()`
   - Transition constants: `crate::pipeline::RECORDING`, `crate::pipeline::PROCESSING`, `crate::pipeline::IDLE`
   - Spawn pipeline: `tauri::async_runtime::spawn(async move { crate::pipeline::run_pipeline(app_clone).await; });`
   - Reset to idle on discard: `crate::pipeline::PipelineState` `.set(crate::pipeline::IDLE)` and update tray via `crate::tray::set_tray_state()`

   This keeps the dependency direction clean: pipeline depends on vad (via use-import), vad references pipeline only through inline full paths.

3. Add `mod vad;` to lib.rs (after `mod pipeline;`, before the `#[cfg(feature = "whisper")]` block).

**Important:** Do NOT call `vad.predict()` inside any cpal audio callback. The VAD worker runs on its own async task, reading the shared buffer. Do NOT share VoiceActivityDetector across threads — create fresh per session.

**Build note:** First build will download ONNX Runtime binaries from pyke's CDN (~50-100MB). Requires internet access. Subsequent builds use cached binary. If build fails with download/TLS errors, retry with internet access.
