**Refactor src-tauri/src/transcribe.rs:**

1. **Add ModelMode enum:**
   ```rust
   pub enum ModelMode { Gpu, Cpu }
   ```

2. **Add detect_gpu() -> ModelMode** function:
   - Try Nvml::init()
   - If Ok, try nvml.device_by_index(0)
   - If device found, log "NVIDIA GPU detected: {device.name()}" and return ModelMode::Gpu
   - If Nvml::init() fails, log "NVML init failed (no NVIDIA GPU): {error}" and return ModelMode::Cpu
   - If device_by_index fails, log "NVML init OK but no device: {error}" and return ModelMode::Cpu

3. **Refactor resolve_model_path(mode: &ModelMode) -> Result<PathBuf, String>:**
   - Match on mode:
     - Gpu: look for `ggml-large-v3-turbo-q5_0.bin`
     - Cpu: look for `ggml-small.en-q5_1.bin`
   - Error message includes the correct download URL for the selected model
   - Log which model was selected and why

4. **Refactor load_whisper_context(model_path: &str, mode: &ModelMode) -> Result<WhisperContext, String>:**
   - Set use_gpu(true) for Gpu mode, use_gpu(false) for Cpu mode
   - Log "Loading whisper model: {path} (GPU={true/false})"

5. **Update lib.rs setup()** to use the new API:
   ```rust
   let mode = transcribe::detect_gpu();
   let model_path = transcribe::resolve_model_path(&mode)?;
   let ctx = transcribe::load_whisper_context(model_path.to_str().unwrap(), &mode)?;
   ```

6. **Add Tauri command force_cpu_transcribe(path: String)** for testing CPU fallback:
   - Load the CPU model (ggml-small.en-q5_1.bin) with use_gpu(false)
   - Run transcription on the provided WAV file
   - Log duration — CPU inference on small model should take 2-10s for a 5s clip (acceptable per success criteria)
   - This command is for Phase 2 verification only; it will be removed or hidden in later phases

**Important:** The force_cpu_transcribe command creates a temporary WhisperContext with use_gpu(false) to verify CPU path works even on a machine that has a GPU. This is the only way to test CORE-04 on the dev machine (which has a P2000).
