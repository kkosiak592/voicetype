<decisions_made>

- CUDA 12.9 (not 13.1): CUDA 13.x dropped sm_61 (Pascal) support. CUDA 12.9 is the last version supporting P2000's compute capability 6.1
- CUDA 12.9 (not 11.7): MSVC 14.44 (VS 2022 17.14) STL headers incompatible with CUDA 11.7's nvcc
- whisper-rs 0.15.x API: full_get_segment_text replaced by get_segment(i).to_str(), full_n_segments returns c_int directly (not Result)
- tokio removed: replaced tokio::task::spawn_blocking with std::thread::spawn + mpsc channel for blocking inference
- Bindgen needs 3 include paths on Windows: clang builtins, Windows UCRT (stdio.h), MSVC headers (vcruntime.h)
</decisions_made>

<context>
The CUDA build environment saga is resolved. Key learnings:
1. CUDA_PATH_V12_9 env var (not registry key) is what MSBuild .targets reads
2. VS integration files must be manually copied to BuildTools/BuildCustomizations after every CUDA install
3. Bindgen's BINDGEN_EXTRA_CLANG_ARGS splits on whitespace — paths with spaces need inner quotes for CommandLineToArgvW parsing
4. CUDA bin/ must be in PATH at runtime for DLL loading (cudart64_12.dll, cublas64_12.dll)
5. Bash shell doesn't inherit Windows system env vars or see audio devices — PowerShell needed for full app testing
