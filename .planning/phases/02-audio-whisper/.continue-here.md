<context>
The bash shell used by Claude Code does not inherit Windows system env vars, so all env vars (CUDA_PATH, LIBCLANG_PATH, CMAKE_CUDA_ARCHITECTURES) must be explicitly exported in each build command. The cargo PATH also needs to be added manually.

The CUDA installer has a persistent bug where it doesn't properly install VS integration files into BuildTools (only into Community, or not at all). Manual copy from CUDA extras/visual_studio_integration/MSBuildExtensions/ is required. Similarly, the registry InstallDir value ends up empty, needing manual Set-ItemProperty.

whisper-rs-sys v0.14.1 uses cmake with "Visual Studio 17 2022" generator, which requires the CUDA .props/.targets/.xml files in the BuildCustomizations directory AND the registry InstallDir to be set.
