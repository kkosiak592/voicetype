<objective>
Establish the cryptographic signing foundation and public distribution backend for VoiceType auto-updates.

This plan generates an Ed25519 keypair required by tauri-plugin-updater, pushes source to a public GitHub repository (required for unauthenticated release asset access), and stores the private key as a GitHub Actions secret so CI/CD pipelines can sign release artifacts.

Key constraint from STATE.md: private key loss is irreversible — existing installs can never receive updates if the key is lost. A mandatory human checkpoint backs up the key before it is used anywhere.
</objective>

```bash
# Generate a strong random password
openssl rand -base64 32 > "$HOME/.voicetype-signing.password"
chmod 600 "$HOME/.voicetype-signing.password"

cd /c/Users/kkosiak.TITANPC/Desktop/Code/voice-to-text

  --password "$SIGNING_PASSWORD" \
  --write-keys "$HOME/.voicetype-signing.key" \
```

If the repo already exists or that command fails, use:
```bash

```

Step 3 — Set GitHub Actions secrets (pipe via variable, not shell args, to avoid process list exposure):

```bash
PRIVATE_KEY=$(cat "$HOME/.voicetype-signing.key")
SIGNING_PASSWORD=$(cat "$HOME/.voicetype-signing.password")

gh secret set TAURI_SIGNING_PRIVATE_KEY \
  --body "$PRIVATE_KEY" \
  --repo kkosiak592/voicetype

gh secret set TAURI_SIGNING_PRIVATE_KEY_PASSWORD \
  --body "$SIGNING_PASSWORD" \
  --repo kkosiak592/voicetype
```

Step 4 — Verify signing round-trip using the public key embedded in tauri.conf.json:

```bash
cd /c/Users/kkosiak.TITANPC/Desktop/Code/voice-to-text
SIGNING_PASSWORD=$(cat "$HOME/.voicetype-signing.password")

# Create test artifact
echo "voicetype signing verification test" > /tmp/voicetype-test.bin

# Sign it
npx tauri signer sign \
  --private-key-path "$HOME/.voicetype-signing.key" \
  --password "$SIGNING_PASSWORD" \
  /tmp/voicetype-test.bin
# This produces /tmp/voicetype-test.bin.sig

# Verify: check the .sig file was produced
test -f /tmp/voicetype-test.bin.sig && echo "Signature file created: OK"

# Extract pubkey from tauri.conf.json and verify
PUBKEY=$(node -e "const c=JSON.parse(require('fs').readFileSync('src-tauri/tauri.conf.json','utf8')); process.stdout.write(c.app.updater.pubkey)")
echo "Pubkey from config: $PUBKEY"

# Tauri 2 sign command also accepts --verify for round-trip check
# If --verify flag not available, the .sig file existing with non-zero size is sufficient
ls -la /tmp/voicetype-test.bin.sig
```

Step 5 — Commit the tauri.conf.json change (the public key is safe to commit — it's public by design):
