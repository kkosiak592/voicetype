**Part A — Wire dispatch_hook_event to handle_shortcut (keyboard_hook.rs):**

The dispatcher thread in `install()` currently logs events. Replace the `dispatch_hook_event` function to call `handle_shortcut()` with a synthetic ShortcutEvent.

The challenge: `ShortcutEvent` is a struct from tauri-plugin-global-shortcut. Check if it has public constructors or if its fields are public. If not constructible directly, use an alternative approach:

**Preferred approach:** Instead of constructing ShortcutEvent, refactor handle_shortcut to accept an enum that both the global-shortcut handler and the hook dispatcher can produce. Create a thin wrapper:

In `lib.rs`, extract the body of `handle_shortcut` into a new function `handle_hotkey_event(app: &tauri::AppHandle, pressed: bool)` that takes a simple bool (true=pressed, false=released). Then have both `handle_shortcut` and the hook dispatcher call `handle_hotkey_event`.

```rust
// lib.rs — new function (extract from handle_shortcut body)
pub(crate) fn handle_hotkey_event(app: &tauri::AppHandle, pressed: bool) {
    // Move the entire match body from handle_shortcut here
    // pressed=true -> the Pressed branch
    // pressed=false -> the Released branch
}

fn handle_shortcut(app: &tauri::AppHandle, event: &tauri_plugin_global_shortcut::ShortcutEvent) {
    use tauri_plugin_global_shortcut::ShortcutState;
    match event.state {
        ShortcutState::Pressed => handle_hotkey_event(app, true),
        ShortcutState::Released => handle_hotkey_event(app, false),
    }
}
```

Then in `keyboard_hook.rs`, update `dispatch_hook_event`:
```rust
fn dispatch_hook_event(app: &tauri::AppHandle, event: HookEvent) {
    match event {
        HookEvent::Pressed => crate::handle_hotkey_event(app, true),
        HookEvent::Released => crate::handle_hotkey_event(app, false),
    }
}
```

This avoids constructing private ShortcutEvent types and creates a clean internal API.

**Part D — Conditional hook install in setup() (lib.rs):**

Add a managed state for the hook handle:

Register it on the Builder (before setup):


In setup(), after all existing state registration but BEFORE the global-shortcut plugin registration block (line ~1549), add conditional routing:

```rust
#[cfg(windows)]
let hook_active = if is_hook_hotkey(&hotkey) {
    match keyboard_hook::install(app.handle().clone()) {
        Ok(handle) => {
            log::info!("Keyboard hook installed for hotkey: {}", hotkey);
            let hook_state = app.state::<HookHandleState>();
            let mut guard = hook_state.0.lock().unwrap_or_else(|e| e.into_inner());
            *guard = Some(handle);
            true
        }
        Err(e) => {
            log::error!("Failed to install keyboard hook: {} — falling back to global shortcut", e);
            false
        }
    }
} else {
    false
};

#[cfg(not(windows))]
let hook_active = false;
```

Then modify the global-shortcut registration to be conditional:
```rust
#[cfg(desktop)]
if !hook_active {
    app.handle().plugin(
        tauri_plugin_global_shortcut::Builder::new()
            .with_shortcuts([hotkey.as_str()])?
            .with_handler(|app, _shortcut, event| {
                handle_shortcut(app, &event);
            })
            .build(),
    )?;
}
```

This means: if hotkey is "ctrl+win" AND hook installed successfully, skip global-shortcut. If hotkey is standard or hook failed, use global-shortcut as before.
