Create `RELEASING.md` at the repo root with a complete, unambiguous release runbook.

**Structure:**

1. **Header** — "Releasing VoiceType" with a one-line summary: version bump, commit, tag, push, CI handles the rest.

2. **Prerequisites** — What must be true before cutting a release:
   - Clean working tree (`git status` shows nothing)
   - On the `master` branch
   - All changes for the release are merged

3. **Version Bump** — Exact instructions to update version in all 3 files:
   - `package.json` line 4: `"version": "X.Y.Z"`
   - `src-tauri/Cargo.toml` line 3: `version = "X.Y.Z"`
   - `src-tauri/tauri.conf.json` line 4: `"version": "X.Y.Z"`
   - Include a note: all three MUST match, and MUST match the tag (without the `v` prefix)

4. **Update Changelog** — Instructions to update CHANGELOG.md:
   - Move entries from `[Unreleased]` to a new `[X.Y.Z]` section with today's date
   - Add a fresh empty `[Unreleased]` section at top
   - Note: the release notes from the changelog entry will be used for the GitHub Release description

5. **Commit** — Exact command:
   ```
   git add package.json src-tauri/Cargo.toml src-tauri/tauri.conf.json CHANGELOG.md
   git commit -m "release: vX.Y.Z"
   ```
   Use conventional commit format with `release:` prefix.

6. **Tag** — Use annotated tags:
   ```
   git tag -a vX.Y.Z -m "vX.Y.Z"
   ```
   Annotated tags (not lightweight) because they store tagger info and work better with `git describe`.

7. **Push** — Push commit and tag together:
   ```
   git push origin master --follow-tags
   ```
   This triggers the GitHub Actions release workflow.

8. **What Happens Next** — Brief description of the CI pipeline:
   - GitHub Actions builds the NSIS installer on Windows
   - Signs with Ed25519 key from repo secrets
   - Generates latest.json for auto-updater
   - Publishes a GitHub Release with installer, signature, and latest.json
   - Existing installs will see the update on next launch

9. **Versioning** — Semver rules for this project:
   - MAJOR: Breaking changes to user-facing behavior or data formats
   - MINOR: New features, new settings, new capabilities
   - PATCH: Bug fixes, performance improvements, dependency updates
   - Pre-1.0 (current): MINOR for features, PATCH for fixes — no stability guarantees yet

10. **Troubleshooting** — Common issues:
    - "CI didn't trigger" — verify tag matches `v*` pattern, check Actions tab
    - "Version mismatch" — all 3 files and the tag must agree
    - "Signing failed" — check TAURI_SIGNING_PRIVATE_KEY secret is set in repo settings

**Formatting:**
- Use Markdown with clear numbered steps
- Include the actual shell commands (copy-pasteable)
- Keep it concise — a runbook, not a tutorial
- Use backtick code blocks for all commands and file references

Create `CHANGELOG.md` at the repo root following the [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) convention.

**Structure:**

1. **Header:**
   ```markdown
   # Changelog

   All notable changes to VoiceType will be documented in this file.

   The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
   and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).
   ```

2. **Unreleased section** — Empty template for accumulating changes:
   ```markdown
   ## [Unreleased]
   ```

3. **v0.1.0 initial release entry** — Document what shipped in v1.0 MVP as the baseline:
   ```markdown
   ## [0.1.0] - 2026-03-02

   ### Added
   - Local voice-to-text transcription with zero internet dependency
   - Dual engine support: Whisper (CUDA) and Parakeet TDT (CUDA/DirectML)
   - Hold-to-talk and toggle recording modes with global hotkey (Ctrl+Shift+Space default)
   - Silero VAD silence detection for automatic stop in toggle mode
   - Glassmorphism pill overlay with real-time frequency visualization
   - Vocabulary profiles with custom word dictionaries
   - Structural engineering domain vocabulary profile
   - First-run setup wizard with GPU auto-detection and model download
   - System tray with microphone state indicator
   - Settings: hotkey rebinding, theme toggle, autostart, model selection
   - NSIS installer for Windows distribution
   - Auto-update infrastructure (updater plugin, Ed25519 signing)
   ```

4. **Footer links** — Comparison links for GitHub:
   ```markdown
   [Unreleased]: https://github.com/kkosiak592/voicetype/compare/v0.1.0...HEAD
   [0.1.0]: https://github.com/kkosiak592/voicetype/releases/tag/v0.1.0
   ```

**Categories used** (Keep a Changelog standard):
- `Added` — New features
- `Changed` — Changes in existing functionality
- `Fixed` — Bug fixes
- `Removed` — Removed features

Only include categories that have entries (don't add empty sections).

**When adding a new release**, the pattern is:
1. Create new `## [X.Y.Z] - YYYY-MM-DD` section
2. Move items from Unreleased into it under appropriate categories
3. Update the `[Unreleased]` comparison link: `compare/vX.Y.Z...HEAD`
4. Add the new version's link: `[X.Y.Z]: https://github.com/kkosiak592/voicetype/releases/tag/vX.Y.Z`
