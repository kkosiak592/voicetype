# Phase 19.3 Plan 02: Profile Simplification Summary

**Multi-profile system removed; single editable vocabulary prompt field replaces profile switcher with zero functionality regression for corrections dictionary and ALL CAPS toggle**

## Accomplishments

- Simplified `profiles.rs`: removed `id`, `name` fields and all hardcoded profile functions (`structural_engineering_profile`, `general_profile`, `get_all_profiles`); added `default_profile()`
- Removed `ProfileInfo` struct, `get_profiles`, `set_active_profile` IPC commands from `lib.rs`
- Added `get_vocabulary_prompt`, `set_vocabulary_prompt`, `get_all_caps` IPC commands
- Updated `save_corrections` to use `corrections.default` flat key (no profile scoping)
- Updated `set_all_caps` to use flat `all_caps` key (no profile scoping)
- Added migration logic in `setup()`: `corrections.structural-engineering` -> `corrections.default`, `profiles.structural-engineering.all_caps` -> `all_caps`
- Created `VocabularySection.tsx` with vocabulary prompt textarea (1s debounce + onBlur save), ALL CAPS toggle, and corrections dictionary editor
- Deleted `ProfileSwitcher.tsx` and `ProfilesSection.tsx`
- Updated `Sidebar.tsx`: `'profiles'` -> `'vocabulary'`; added `'history'` section (placeholder for Plan 03)
- Removed `activeProfile` state from `App.tsx` and `AppSettings`/`DEFAULTS` from `store.ts`

## Decisions Made

- `corrections.default` flat key replaces `corrections.{profile_id}` — no profiles means no per-profile scoping needed
- `all_caps` flat key replaces `profiles.{profile_id}.all_caps` — single global setting
- 1-second debounce plus onBlur save for vocabulary prompt — avoids IPC on every keystroke while ensuring persistence on focus leave
- Migration runs in `setup()` reading disk state before SettingsState is populated from that same disk — checks if target keys absent and source keys present, copies value and flushes immediately
- `history` SectionId added to Sidebar in this plan (not Plan 03) to prevent file conflict between waves
