**3. Rewrite `src/components/FrequencyBars.tsx` with RAF sinusoidal loop:**
Per CONTEXT.md: "Smooth wave bars with sinusoidal interpolation instead of random jitter per render. Gradient per bar: indigo at base, lighter purple at tip. Remove the red recording dot."

Keep the same `level` prop interface (0-1 normalized RMS). Remove `barCount` prop — use fixed 12 bars (Claude's discretion on count). Remove `BAND_MULTIPLIERS`. Remove `Math.random()` jitter.

Use `requestAnimationFrame` + `useRef` for the animation loop. Each bar gets a unique frequency (1.2-3.0 Hz range) and phase offset (spread across 2pi). Amplitude follows bell curve (center bars taller). Heights composite: sinusoidal wave scaled by `level` prop, plus small idle wave (0.08 amplitude) so bars are never fully flat.

Bar styling: `width: 3px`, `border-radius: full`, `background: linear-gradient(to top, #6366f1, #c084fc)` (indigo base, lighter purple tip). Add `transition: height 40ms ease-out` for micro-smoothing between RAF frames.

Container: `flex items-end gap-[2px]`, fixed height of 22px.

The RAF loop must:
- Store `startTime` in a `useRef` (not state)
- Use `performance.now()` for time base
- Access `level` via a `useRef` updated in a separate `useEffect` (avoid adding `level` to RAF effect deps which would restart the loop on every level update)
- Cancel via `cancelAnimationFrame(rafRef.current)` in cleanup

The component should NOT manage its own visibility — Pill.tsx conditionally renders it. When unmounted, React cleanup fires and cancels the RAF loop.
