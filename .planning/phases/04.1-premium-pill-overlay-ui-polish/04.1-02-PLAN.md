  <action>
**Rewrite src/Pill.tsx** with these changes. Preserve: drag handling (handleMouseDown/handleMouseUp with focusable toggle), position persistence via tauri-plugin-store, all five event listeners. Change everything else.

**1. Add animation state tracking:**
Add an `animState` ref or state alongside `displayState`:
- `animState`: `"hidden" | "entering" | "visible" | "exiting"` — controls entrance/exit CSS classes
- Keep `displayState`: `"hidden" | "recording" | "processing" | "success" | "error"` — controls content

**2. Rewrite pill-show handler:**
When `pill-show` fires:
- Clear any pending hide timer
- Call `appWindow.show()`
- Set `animState` to `"entering"`
- After the entrance animation completes (listen for `animationend` on the container, or use a 220ms timeout matching the pill-enter duration), set `animState` to `"visible"`

**3. Rewrite pill-hide handler:**
When `pill-hide` fires:
- Set `animState` to `"exiting"`
- After the exit animation completes (180ms timeout matching pill-exit duration), call `appWindow.hide()` and set both `animState` to `"hidden"` and `displayState` to `"hidden"`
- Do NOT set displayState to hidden before the exit animation plays — that would clear the content during the scale-down

**4. Update pill-result handler for success/error:**
- Success: set `displayState` to `"success"`, wait 600ms (checkmark draws in 280ms + 320ms hold), then trigger exit animation (set `animState` to `"exiting"`, after 180ms hide window)
- Error: set `animState` to `"exiting"` immediately (silent dismiss per CONTEXT.md — "pill fades away without drawing attention to failed attempts"). After 180ms hide window. Do NOT set displayState to "error" visually — just exit.

**5. Update container div:**
Replace the old classes with:
```tsx
<div
  ref={containerRef}
  onMouseDown={handleMouseDown}
  onMouseUp={handleMouseUp}
  className={`
    pill-glass
    w-[160px] h-[48px] rounded-full
    flex items-center justify-center
    cursor-grab active:cursor-grabbing
    select-none
    ${animState === "entering" ? "pill-entering" : ""}
    ${animState === "exiting" ? "pill-exiting" : ""}
    ${animState === "hidden" ? "opacity-0 pointer-events-none" : ""}
    ${displayState === "processing" ? "pill-processing" : ""}
  `}
>
```

Key changes from old: `pill-glass` replaces `bg-black/75`. Width 160px, height 48px. No `transition-opacity duration-300` (entrance/exit keyframes handle it). No `pill-success` or `pill-error` classes (those are deleted).

**6. Update content rendering:**
- Recording: `<FrequencyBars level={level} />` — NO red dot (per CONTEXT.md: "Remove the red recording dot"). Center the bars in the pill with appropriate padding.
- Processing: `<ProcessingDots />` — NO text. Add import.
- Success: `<CheckmarkIcon />` — NO text. Add import.
- Error: render nothing or the same content as previous state — the pill is already exiting via scale-down.

**7. Cross-fade for recording->processing transition (per CONTEXT.md: "bars shrink down while pulsing glow fades in, no content flash"):**
The simplest approach: wrap each content block in a div with the `pill-content-fade-in` class when it appears. Since React unmounts the previous and mounts the new in the same render, the new content fading in from opacity 0 over 120ms prevents the flash. The glow pulse on the container (`pill-processing` class) provides the processing visual immediately.

**8. Update position initialization:**
Change the centering calculation from `(screenW - 120) / 2` to `(screenW - 160) / 2` and `screenH - 48 - 60` for the new dimensions.

**9. Update `src-tauri/tauri.conf.json`:**
Change pill window from `"width": 120, "height": 40` to `"width": 160, "height": 48`.
