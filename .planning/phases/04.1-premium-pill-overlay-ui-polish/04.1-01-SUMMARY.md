## Accomplishments

- Rewrote pill.css with glassmorphism base class, spring entrance/exit animations, indigo glow pulse processing state, dot-bounce keyframe, draw-check keyframe, and content cross-fade helpers. Removed all deprecated @property/conic-gradient/border-spin/success-flash/error-flash patterns.

- Created CheckmarkIcon component with SVG polyline that draws itself via stroke-dashoffset animation using pill-checkmark-draw CSS class.
- Rewrote FrequencyBars with a requestAnimationFrame loop, per-bar sinusoidal frequencies (1.2-3.0 Hz), per-bar phase offsets spread across 2pi, gaussian bell-curve amplitude envelope, indigo-to-purple gradient, and idle wave so bars are never fully flat.

## Decisions Made

- No `backdrop-filter` in pill.css — confirmed Windows WebView2 transparent window bug #4945 makes it silently fail. Glassmorphism relies on the dark semi-transparent background instead.
- FrequencyBars accesses `level` prop via `useRef` inside the RAF effect (not the prop directly) — avoids adding `level` to the effect deps which would restart the RAF loop on every audio update (~60 times/second).
- Fixed 12 bars, removed `barCount` prop — plan specified removing BAND_MULTIPLIERS and simplifying the interface; 12 provides a clean symmetric waveform.
- Gaussian bell curve for bar amplitude distribution — more mathematically natural than the old hardcoded BAND_MULTIPLIERS array, center bars peak at ~100%, edge bars at ~8%.
- Idle wave amplitude fixed at 0.08 — keeps bars gently animated at level=0 so the component never appears frozen while recording is active but silent.
