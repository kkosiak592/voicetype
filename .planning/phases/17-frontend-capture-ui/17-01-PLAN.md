Modify HotkeyCapture.tsx to support modifier-only combos alongside the existing standard combo capture. All changes are within this single file.

**1. Add `modifierToken` helper function** (before the component, after `normalizeKey`):

```typescript
function modifierToken(code: string): string | null {
  switch (code) {
    case 'ControlLeft':
    case 'ControlRight':
      return 'ctrl';
    case 'MetaLeft':
    case 'MetaRight':
      return 'meta';
    case 'AltLeft':
    case 'AltRight':
      return 'alt';
    case 'ShiftLeft':
    case 'ShiftRight':
      return 'shift';
    default:
      return null;
  }
}
```

Maps `KeyboardEvent.code` to canonical modifier tokens. Uses `e.code` (not `e.ctrlKey`/`e.metaKey` flags) because on `keyup` the flags reflect the post-release state (already false for the released key).

**Canonical token order:** Define a sort order constant:
```typescript
const MODIFIER_ORDER: Record<string, number> = { ctrl: 0, alt: 1, shift: 2, meta: 3 };
```
When joining tokens from heldRef, sort by this order so the stored combo is deterministic regardless of press order. Example: user presses Win then Ctrl -> heldRef has ["meta", "ctrl"] -> sorted to ["ctrl", "meta"] -> joined as "ctrl+meta".

**2. Add `heldRef` (useRef) and `heldDisplay` (useState)** inside the component:

```typescript
const heldRef = useRef<Set<string>>(new Set());
const [heldDisplay, setHeldDisplay] = useState<string>('');
```

- `heldRef`: Tracks currently held modifier tokens across keydown/keyup events. useRef (not useState) because mutations should not trigger re-renders.
- `heldDisplay`: Drives the progressive display of held modifiers during capture (e.g., "Ctrl + Win...").

**3. Rewrite the keydown/keyup useEffect** (the one guarded by `if (!listening) return`):

Replace the existing keydown-only listener with a dual keydown+keyup approach:

**handleKeyDown:**
- `e.preventDefault()` + `e.stopPropagation()` (same as current)
- Escape check: clear `heldRef.current`, clear `heldDisplay`, set listening false, re-register if value exists (same as current, plus clearing heldRef and heldDisplay)
- Check `modifierToken(e.code)`: if it returns a token, add to `heldRef.current`, update `heldDisplay` with formatted held keys, and `return` (wait for more keys or keyup)
- If `modifierToken` returns null (non-modifier key pressed): clear `heldRef.current`, clear `heldDisplay`, then proceed with existing `normalizeKey(e)` standard combo logic (unchanged from current code)

**handleKeyUp:**
- `e.preventDefault()` + `e.stopPropagation()`
- Check `modifierToken(e.code)`: if null (non-modifier released), clear `heldRef.current`, clear `heldDisplay`, return
- A modifier was released. Read the current set BEFORE deleting: `const tokens = [...heldRef.current].sort((a, b) => (MODIFIER_ORDER[a] ?? 99) - (MODIFIER_ORDER[b] ?? 99))`. Then `heldRef.current.delete(token)`.
- If `heldRef.current.size === 0` (all modifiers released) and `tokens.length > 0`: this is a modifier-only combo. Join tokens: `const combo = tokens.join('+')`. Then:
  - `setListening(false)`, `setError(null)`, `setHeldDisplay('')`
  - If `combo === value`: re-register with `invoke('register_hotkey', { key: value })`, return
  - Otherwise: `await invoke('rebind_hotkey', { old: '', newKey: combo })`, persist to store, call `onChange(combo)`. On error: restore original hotkey, `setError(String(err))`.
- If `heldRef.current.size > 0` (some modifiers still held): update `heldDisplay` with remaining held keys.

**Click-outside handler:** Keep existing logic, but also clear `heldRef.current` and `heldDisplay` on click-away cancel.

**Cleanup function:** In the `return () => { ... }` cleanup, remove both keydown and keyup listeners, and clear `heldRef.current` and `heldDisplay`.

**4. Update the render display:**

Change the listening text from static `'Press a key combo...'` to progressive display:
```tsx
{listening
  ? heldDisplay
    ? `${heldDisplay}...`
    : 'Press a key combo...'
  : formatHotkey(value)}
```

**Critical anti-patterns to avoid:**
- Do NOT use `e.ctrlKey`/`e.metaKey` in the keyup handler to detect which modifier was released — they are already false for the released key.
- Do NOT clear heldRef BEFORE reading it on keyup — read the pre-delete state, then delete.
- Do NOT complete a modifier-only combo on keydown — cannot know if a base key is coming.
- Do NOT forget to clear heldRef on Escape, click-away, and cleanup.
- Do NOT change `normalizeKey()` — it remains correct for the standard path.
- Do NOT change `formatHotkey()` — it already handles "meta" -> "Win" for UI-02.

**Note on `e.repeat`:** Ignore repeated keydown events (where `e.repeat === true`) to avoid re-adding already-tracked modifiers. Add `if (e.repeat) return;` at the top of handleKeyDown after preventDefault.
