# Phase 19.1 Plan 01: Moonshine Tiny Backend Integration Summary

**One-liner:** Moonshine Tiny ONNX engine wired as third TranscriptionEngine with VAD chunking, GPU provider selection, multi-file download, and full IPC handler coverage.

## What Was Built

Complete Rust backend integration for Moonshine Tiny as a selectable transcription engine in VoiceType, following the established Parakeet pattern:

### Task 1: Engine Infrastructure

**Cargo.toml:** Added `moonshine = ["dep:transcribe-rs", "dep:ort"]` feature. Updated `default` features to include `moonshine`. The `dep:ort` inclusion was required because `transcribe_moonshine.rs` directly uses `ort::execution_providers::*` types for CUDA EP construction — the `transcribe-rs` re-export path alone is insufficient.

**lib.rs (enum + state):**
- `TranscriptionEngine::Moonshine` variant added — serializes as `"moonshine"` via `kebab-case`
- `MoonshineStateMutex` struct added following the exact `ParakeetStateMutex` pattern: `Mutex<Option<Arc<Mutex<MoonshineEngine>>>>` for thread-safe, clonable, swap-at-runtime model state
- `mod transcribe_moonshine` gated with `#[cfg(feature = "moonshine")]`

**transcribe_moonshine.rs (new file):**
- `load_moonshine(model_dir, provider)` — creates `MoonshineModelParams::tiny()`, sets CUDA EP when provider is "cuda" (CPU fallback for DirectML pending validation), calls `TranscribeRsEngine::load_model_with_params`
- `warm_up_moonshine(engine)` — 8000 zero-sample inference to initialize CUDA context
- `transcribe_with_moonshine(engine, audio)` — checks 30s limit, splits via `vad_chunk_for_moonshine` if over 30s, concatenates chunk results with space separator

**vad.rs:** Added `vad_chunk_for_moonshine(samples) -> Vec<Vec<f32>>` — port of `benchmark.rs::vad_chunk_audio` using `log::info!` instead of `println!`, fresh VAD per call, same SILENCE_SPLIT_CHUNKS/MAX_SEGMENT/MIN_SEGMENT constants.

**pipeline.rs:** Added `TranscriptionEngine::Moonshine` dispatch arm after Parakeet, following identical pattern: clone `Arc` from `MoonshineStateMutex`, `spawn_blocking`, call `transcribe_with_moonshine`, handle all error arms.

### Task 2: IPC Wiring

**download.rs:**
- `MOONSHINE_TINY_FILES` constant array (3 files: encoder, decoder_merged, tokenizer)
- `moonshine_tiny_model_dir()` — returns `models_dir().join("moonshine-tiny-ONNX")`
- `moonshine_tiny_model_exists()` — checks for `decoder_model_merged.onnx` sentinel
- `moonshine_download_url(filename)` — constructs HuggingFace URL
- `download_moonshine_tiny_model` Tauri command — replicates `download_parakeet_fp32_model` pattern with 3-file cumulative progress, cleanup on error

**lib.rs IPC changes:**
- `set_engine`: handles `"moonshine"` string → `TranscriptionEngine::Moonshine`; adds Moonshine loading block (load model with GPU provider, register in MoonshineStateMutex, spawn warm-up thread, fall back to Whisper on error)
- `read_saved_engine`: handles `Some("moonshine")` from settings.json
- `get_engine`: returns `"moonshine"` for Moonshine variant
- `get_gpu_info`: handles Moonshine in `engine_str` match, returns `"moonshine-tiny"` as `active_model`, uses `parakeet_provider` for EP
- `list_models`: adds Moonshine Tiny entry (id: `moonshine-tiny`, 108 MB, not recommended/GPU-only)
- `check_first_run`: adds `moonshine_exists` check to `needs_setup` condition
- **Builder**: registers `MoonshineStateMutex(Mutex::new(None))` before `run()` under `#[cfg(feature = "moonshine")]`
- **invoke_handler**: adds `download::download_moonshine_tiny_model`
- **Startup loading**: adds Moonshine block after Parakeet block — loads model if `saved_engine == Moonshine`, falls back to Whisper on error or missing model dir

### Auto-fixed Issues

**1. [Rule 2 - Missing critical functionality] Added `dep:ort` to moonshine feature**
- **Found during:** Task 1 — cargo check error on `ort::execution_providers::CUDAExecutionProvider`
- **Issue:** Plan specified `moonshine = ["dep:transcribe-rs"]` but `transcribe_moonshine.rs` uses `ort::execution_providers::*` types directly. Without `dep:ort`, the `ort` crate types are not accessible.
- **Fix:** Changed `moonshine = ["dep:transcribe-rs"]` to `moonshine = ["dep:transcribe-rs", "dep:ort"]`
- **Files modified:** `src-tauri/Cargo.toml`
- **Commit:** `1eedc10`

**2. [Rule 1 - Bug] Added TranscribeRsEngine trait import in transcribe_moonshine.rs**
- **Found during:** Task 1 — cargo check errors "method not found" for `load_model_with_params` and `transcribe_samples`
- **Issue:** In Rust 2021 edition, trait methods require the trait to be in scope. `transcribe_rs::TranscriptionEngine` trait was not imported, so `engine.load_model_with_params()` and `engine.transcribe_samples()` were not found.
- **Fix:** Added `use transcribe_rs::TranscriptionEngine as TranscribeRsEngine;` and used UFCS syntax (`TranscribeRsEngine::load_model_with_params(&mut engine, ...)`)
- **Files modified:** `src-tauri/src/transcribe_moonshine.rs`
- **Commit:** `1eedc10`

**3. [Rule 1 - Bug] Fixed non-exhaustive match errors in get_engine and get_gpu_info**
- **Found during:** Task 1 — cargo check errors for `TranscriptionEngine::Moonshine` not covered in existing match blocks
- **Issue:** Adding `Moonshine` variant to `TranscriptionEngine` enum required updating all existing match expressions
- **Fix:** Added `Moonshine` arms to `get_engine`, `get_gpu_info` (engine_str and active_model matches)
