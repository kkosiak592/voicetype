## Decisions Made

- Used `Image::from_bytes(ICON_IDLE)?` in `build_tray()` instead of `default_window_icon()` to match what `set_tray_state()` uses at runtime. When the two calls use different sources, Windows creates two HICON handles registered as separate notification area entries.
- Generated tray PNG files using pure Node.js (signature + IHDR + deflate-compressed IDAT + IEND) without external image libraries — viable since the microphone is a simple geometric shape renderable at pixel coordinates.
- `npx tauri icon` in version 2.10.0 accepts SVG input directly — no pre-conversion to PNG needed.
