    **Fix 1: Replace `default_window_icon()` with `Image::from_bytes(ICON_IDLE)` in `build_tray()`**

    In `build_tray()`, line 50 currently reads:
    ```rust
    .icon(app.default_window_icon().unwrap().clone())
    ```

    Replace with:
    ```rust
    .icon(tauri::image::Image::from_bytes(ICON_IDLE)?)
    ```

    This eliminates the duplicate icon bug. The `default_window_icon()` call reads from the bundle icon files (producing HICON handle A), while `set_tray_state()` uses `Image::from_bytes()` from the embedded PNG (producing HICON handle B). Windows treats these as two separate notification area entries. Using the same `ICON_IDLE` bytes for both initial and runtime icons ensures a single entry.

    Use `?` error propagation (not `.unwrap()`) — `build_tray()` already returns `tauri::Result<()>`.

    **Fix 2: Add tooltip to TrayIconBuilder**

    Add `.tooltip("VoiceType - Idle")` to the builder chain in `build_tray()`:
    ```rust
    TrayIconBuilder::with_id("tray")
        .icon(tauri::image::Image::from_bytes(ICON_IDLE)?)
        .tooltip("VoiceType - Idle")
        .menu(&menu)
        ...
    ```

    **Fix 3: Add tooltip updates to `set_tray_state()`**

    Refactor `set_tray_state()` to update both icon and tooltip on every state change:

    ```rust
    pub fn set_tray_state(app: &tauri::AppHandle, state: TrayState) {
        let (icon_bytes, tooltip) = match state {
            TrayState::Idle       => (ICON_IDLE,       "VoiceType - Idle"),
            TrayState::Recording  => (ICON_RECORDING,  "VoiceType - Recording"),
            TrayState::Processing => (ICON_PROCESSING, "VoiceType - Processing"),
        };
        if let Some(tray) = app.tray_by_id("tray") {
            if let Ok(image) = tauri::image::Image::from_bytes(icon_bytes) {
                let _ = tray.set_icon(Some(image));
            }
            let _ = tray.set_tooltip(Some(tooltip));
        }
    }
    ```

    Keep silent failure pattern (`if let`, `let _ =`) for runtime state updates — tray icon is non-critical feedback. The existing pattern of `if let Some(tray) = app.tray_by_id("tray")` is correct and stays.

    **Do NOT:**
    - Add `app.trayIcon` to tauri.conf.json (causes second tray icon per GitHub issue #8982)
    - Change how `build_tray()` is called in lib.rs (it's called once in setup(), which is correct)
    - Change the `Manager` import or other existing imports unless needed
