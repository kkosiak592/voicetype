# Phase 06.1: Fix duplicate tray icons and replace default square icon with proper app icon - Context

**Gathered:** 2026-02-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix two tray icon bugs: (1) eliminate duplicate icons appearing in the Windows system tray, and (2) replace the default Tauri square icon with a proper VoiceType app icon. Apply the new icon consistently across all Windows surfaces (tray, taskbar, title bar, installer).

</domain>

<decisions>
## Implementation Decisions

### App Icon
- Need to create a new custom VoiceType icon — no icon exists yet
- Icon should be used everywhere: system tray, taskbar, window title bar, and installer (bundle icons)
- Replace all Tauri default icons (32x32.png, 128x128.png, 128x128@2x.png, icon.ico, icon.icns) with the new design
- Generate all required sizes for Windows surfaces

### Duplicate Tray Icons
- User sees both issues: two icons appearing simultaneously on start AND ghost icons persisting after crash/restart
- Observed in dev mode (cargo tauri dev) — production not yet tested
- One icon is purple, the other is light grey — confirms two different icon sources: `build_tray()` uses `app.default_window_icon()` (Tauri default grey square) while state transitions apply the colored PNGs (tray-idle.png etc.)
- Root cause likely: `default_window_icon()` at `tray.rs:50` creates initial icon from bundle defaults, then `set_tray_state()` switches to the PNG state icons — but both icon registrations may persist as separate tray entries
- Ghost icons after crash are standard Windows behavior (Explorer caches tray icons until hover)

### Tray Tooltip
- Add tooltip showing app name + current state: "VoiceType - Idle" / "VoiceType - Recording" / "VoiceType - Processing"
- Tooltip should update when tray state changes

### Claude's Discretion
- Icon design (microphone, waveform, speech bubble, or combination — whatever fits the app's voice-to-text purpose)
- Icon color scheme
- Whether to redesign tray state icons (tray-idle.png, tray-recording.png, tray-processing.png) to match the new app icon or keep existing
- Tray icon sizes (follow Tauri/Windows best practices)

</decisions>

<specifics>
## Specific Ideas

- One duplicate icon is purple, the other is light grey — fix should ensure only one icon instance exists in the tray at any time
- Ghost icons after crash are acceptable Windows behavior (no code fix needed) — the real fix is preventing two simultaneous icons

</specifics>

## Existing Code Insights

### Reusable Assets
- `tray.rs` — Complete tray module with `build_tray()` and `set_tray_state()` functions
- Three tray state PNGs already exist: `tray-idle.png`, `tray-recording.png`, `tray-processing.png` (embedded via `include_bytes!`)
- `tray-icon.png` also exists in icons/ (unclear purpose — may be unused)

### Established Patterns
- Tray icon state changes use `app.tray_by_id("tray")` lookup pattern
- Icons embedded at compile time via `include_bytes!` for state PNGs
- Initial icon uses `app.default_window_icon().unwrap().clone()` — this is the source of the default square icon bug
- Bundle icons in `tauri.conf.json` under `bundle.icon` array

### Integration Points
- `build_tray()` called once in `lib.rs:494` during `setup()`
- `set_tray_state()` called from hotkey handlers in `lib.rs` (both initial handler and `rebind_hotkey`)
- `tauri.conf.json` `bundle.icon` array needs updating with new icon files
- Tauri features: `tray-icon` and `image-png` already enabled in Cargo.toml
