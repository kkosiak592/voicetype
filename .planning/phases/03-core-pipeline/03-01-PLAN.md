   The function implements this exact sequence (per CONTEXT.md locked decisions and RESEARCH.md):
   a. Create arboard::Clipboard instance
   b. Save existing clipboard: `clipboard.get_text().ok()` — returns `Option<String>`, None if non-text content
   c. Set clipboard to the provided text: `clipboard.set_text(text)`
   d. Sleep 75ms (midpoint of 50-100ms range) — mandatory Windows clipboard propagation delay
   e. Simulate Ctrl+V using enigo:
      ```rust
      let mut enigo = Enigo::new(&Settings::default()).map_err(|e| e.to_string())?;
      enigo.key(Key::Control, Press).map_err(|e| e.to_string())?;
      enigo.key(Key::Unicode('v'), Click).map_err(|e| e.to_string())?;
      enigo.key(Key::Control, Release).map_err(|e| e.to_string())?;
      ```
   f. Sleep 120ms (midpoint of 100-150ms range) — wait for target app to consume paste
   g. Restore original clipboard:
      - If saved was `Some(original)`: call `clipboard.set_text(&original)`, log warning on failure per user decision (move on, don't error)
      - If saved was `None`: call `clipboard.set_text("")` to clear (arboard has no clear())

   Use imports:
   ```rust
   use arboard::Clipboard;
   use enigo::{Direction::{Click, Press, Release}, Enigo, Key, Keyboard, Settings};
   use std::thread;
   use std::time::Duration;
   ```

   Do NOT make inject_text async — it is intentionally synchronous (callers wrap in spawn_blocking).
   Do NOT share Enigo instances across calls — create fresh per invocation per RESEARCH.md anti-patterns.

3. Add `mod inject;` to lib.rs (add it alongside existing mod declarations, NOT behind any feature gate — inject.rs has no whisper dependency).
