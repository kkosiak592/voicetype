Three file changes:

1. **.github/workflows/release.yml** — Insert a new step between "Set CUDA architecture targets" and "Install frontend dependencies":
```yaml
- name: Stage CUDA redistributable DLLs
  shell: bash
  run: |
    mkdir -p src-tauri/cuda-libs
    cp "$CUDA_PATH/bin/cudart64_12.dll"    src-tauri/cuda-libs/
    cp "$CUDA_PATH/bin/cublas64_12.dll"    src-tauri/cuda-libs/
    cp "$CUDA_PATH/bin/cublasLt64_12.dll"  src-tauri/cuda-libs/
    echo "Staged DLL sizes:"
    ls -lh src-tauri/cuda-libs/
```
This runs after CUDA toolkit installs (so $CUDA_PATH exists) and before tauri-action (so bundler can find the files).

2. **src-tauri/tauri.conf.json** — Add a `"resources"` field to the `"bundle"` object, using map syntax with flat filename destinations so DLLs land at install root ($INSTDIR) alongside VoiceType.exe:
```json
"resources": {
  "cuda-libs/cudart64_12.dll": "cudart64_12.dll",
  "cuda-libs/cublas64_12.dll": "cublas64_12.dll",
  "cuda-libs/cublasLt64_12.dll": "cublasLt64_12.dll"
}
```
Place this field after `"createUpdaterArtifacts"` and before `"icon"`. Use MAP syntax (object), NOT array syntax — array syntax preserves subdirectory structure which would put DLLs off the DLL search path.

3. **.gitignore** — Add at the end:
```
# CUDA redistributable DLLs staged at build time (CI-created, not committed)
src-tauri/cuda-libs/
```
