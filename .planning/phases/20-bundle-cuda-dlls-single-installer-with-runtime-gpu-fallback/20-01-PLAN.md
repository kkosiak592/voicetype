<objective>
Bundle three CUDA redistributable DLLs (cudart64_12.dll, cublas64_12.dll, cublasLt64_12.dll) in the NSIS installer via CI workflow and Tauri config changes. Three file changes, all configuration:

**1. `.github/workflows/release.yml`** — Insert a new step named "Stage CUDA redistributable DLLs" AFTER the "Install LLVM/clang" step and BEFORE the "Install frontend dependencies" step (or at minimum before "Build and publish release with Tauri"). The step:

```yaml
- name: Stage CUDA redistributable DLLs
  shell: bash
  run: |
    mkdir -p src-tauri/cuda-libs
    cp "$CUDA_PATH/bin/cudart64_12.dll"    src-tauri/cuda-libs/
    cp "$CUDA_PATH/bin/cublas64_12.dll"    src-tauri/cuda-libs/
    cp "$CUDA_PATH/bin/cublasLt64_12.dll"  src-tauri/cuda-libs/
    echo "Staged CUDA DLL sizes:"
    ls -lh src-tauri/cuda-libs/
```

This copies the three redistributable DLLs from the CUDA Toolkit installation (provided by Jimver/cuda-toolkit) into a staging directory that Tauri's bundler can reference. The `ls -lh` output provides CI log visibility into bundled DLL sizes.

**2. `src-tauri/tauri.conf.json`** — Add a `"resources"` key to the `"bundle"` object using the **map syntax** (object, not array). The map keys are source paths relative to `tauri.conf.json` (i.e., relative to `src-tauri/`), and values are destination filenames at the install root:

```json
"resources": {
  "cuda-libs/cudart64_12.dll": "cudart64_12.dll",
  "cuda-libs/cublas64_12.dll": "cublas64_12.dll",
  "cuda-libs/cublasLt64_12.dll": "cublasLt64_12.dll"
}
```

Place this key inside the `"bundle"` object, after the `"createUpdaterArtifacts"` line and before the `"icon"` array. The flat filename destination (no subdirectory prefix) is critical — Windows DLL search order checks the application directory (where the exe lives) first, so DLLs must land at `$INSTDIR` alongside `VoiceType.exe`, not in a subdirectory.

Do NOT use the array syntax (`"resources": ["cuda-libs/cudart64_12.dll"]`) — that preserves the `cuda-libs/` subdirectory in the installed path, putting DLLs off the implicit search path.

**3. `.gitignore`** — Add the following entry at the end of the file:

```
# CUDA redistributable DLLs staged at build time (CI-created, not committed)
src-tauri/cuda-libs/
```

This prevents accidental commit of 500MB+ CUDA DLLs if someone runs a local build with the staging directory present.
