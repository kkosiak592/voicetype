# Phase 6.2: Premium Waveform Visualization Upgrade - Context

**Gathered:** 2026-03-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the current 24 rainbow DOM-based frequency bars (`FrequencyBars.tsx`) with a Canvas2D neon waveform visualization featuring layered bloom effects inside the pill overlay. Add FFT frequency data from the Rust backend to drive voice-reactive curves. The pill window size (170x38px), state machine, entrance/exit animations, and processing state visuals are unchanged.

</domain>

<decisions>
## Implementation Decisions

### Color palette
- Cyan-to-purple gradient: neon cyan (#06b6d4) core fading to purple (#8b5cf6) edges
- Matches existing processing state palette for cohesive sci-fi aesthetic
- Heavy bloom — thick glow halos, additive blending cranked up, maximum JARVIS energy
- Glow contained within pill boundary (overflow: hidden clips it)

### Waveform shape
- 3 layered overlapping bezier curves at different phase offsets — mesh/ribbon effect
- Bell-curve amplitude envelope — center peaks tallest, edges taper down (proven pattern from current FrequencyBars)

### Backend data
- Add 16-bin FFT frequency data via `rustfft` crate in Rust backend
- Each FFT bin drives a control point on the waveform curves
- Speech formants become visible — voice-reactive frequency visualization

### Idle behavior
- Very dim glow at idle — dramatic contrast when recording activates
- Quick ramp-up transition (~150-200ms ease-out) from idle to full active waveform
- Idle state is a brief transition moment (pill is hidden when not recording/processing)

### Claude's Discretion
- Recording border treatment: keep rainbow or switch to match cyan-purple waveform palette
- Waveform symmetry: mirrored top/bottom vs single-sided
- Audio response feel: smooth amplitude scaling vs punchy elastic easing
- RMS alongside FFT vs FFT-only data protocol
- FFT update rate (30fps vs 60fps)
- Idle animation style (breathing pulse, gentle wave, or flat line)

</decisions>

<specifics>
## Specific Ideas

- JARVIS-style sci-fi aesthetic — the defining reference
- Canvas2D + layered bloom approach (from research: scored 9/10 vs SVG 7/10 vs CSS-only 5/10)
- Additive blending via `globalCompositeOperation: "lighter"` for authentic neon glow
- 3 bloom passes with increasing `shadowBlur` (8, 16, 24) and decreasing `globalAlpha`

</specifics>
