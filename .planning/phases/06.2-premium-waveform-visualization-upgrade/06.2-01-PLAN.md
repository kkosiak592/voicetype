struct PillLevelPayload {
    rms: f32,
    bins: Vec<f32>,
}

**pill.rs — FFT computation function:**
Add `compute_fft_bins(buf: &[f32], n_bins: usize) -> Vec<f32>` function:
- Takes last 256 samples from the buffer
- Applies Hann window inline (formula: `0.5 - 0.5 * cos(2*PI*i/(N-1))`)
- Runs rustfft forward FFT (256-point)
- Takes first 128 magnitudes (meaningful spectrum — second half is mirror)
- Bins into `n_bins` groups (128 / 16 = 8 samples per bin), taking max magnitude per group
- Normalizes all bins to 0-1 range (divide by peak)
- **Gate on RMS:** If `rms < 0.02`, return all-zero bins to avoid visual noise from ambient mic noise (Pitfall 6 from research)
- If buffer has < 256 samples, return `vec![0.0; n_bins]`

**pill.rs — Cache FftPlanner:**
Do NOT create `FftPlanner::new()` on every tick (Pitfall 1). Create the planner and the `Arc<dyn Fft<f32>>` plan ONCE before the async loop starts, then move the `Arc` into the closure. `Arc<dyn Fft<f32>>` is cheap to clone and move.

**pill.rs — Update start_level_stream:**
- Create `FftPlanner::<f32>::new()` and `planner.plan_fft_forward(256)` once before the `while` loop
- In each tick: compute RMS as before, then call `compute_fft_bins` with the buffer, rms, and cached FFT plan
- Change emit from `app.emit_to("pill", "pill-level", level)` to `app.emit_to("pill", "pill-level", PillLevelPayload { rms: level, bins })`
- Change the final zero emission to emit `PillLevelPayload { rms: 0.0, bins: vec![0.0; 16] }`
- Pass the FFT plan (`Arc<dyn Fft<f32>>`) into `compute_fft_bins` to avoid re-planning

**Imports to add:**
```rust

```
Note: `Arc` is already imported from `std::sync` in this file. Use it directly for the FFT plan reference — no alias needed since `Arc` is already in scope.

- Keep 30fps emission rate (research recommendation — RAF interpolates to 60fps on frontend)
- Emit both RMS and bins (research recommendation — RMS gates idle, bins shape waveform)

Bell curve envelope (reuse concept from FrequencyBars):
```typescript
function bellCurve(i: number, count: number): number {
  const x = (i / (count - 1)) * 2 - 1;
  return Math.exp(-x * x * 2.5);
}
const BELL_16 = Array.from({ length: 16 }, (_, i) => bellCurve(i, 16));
```

Control points (16 points across width):
- For each bin `i` (0-15): x = `(i / 15) * W`, y = center ± amplitude
- Amplitude = `amplified * BELL_16[i] * bins[i] * (H/2 * 0.85)` (0.85 to leave margin)
- `amplified = Math.pow(lv, 0.55)` (same non-linear mapping as FrequencyBars)

**3 layered curves (locked decision):**
Draw 3 curves with different phase offsets to create mesh/ribbon effect:
- Curve 1: phase offset 0 (base)
- Curve 2: phase offset `t * 0.7` (slightly shifting)
- Curve 3: phase offset `t * 1.3` (faster shifting)

Each curve: add sinusoidal time-varying component to Y:
```
y = centerY - amplitude * sin(phase + binIndex * 0.4) // top half
```

**Mirrored waveform (research recommendation for Claude's discretion — mirrored looks fuller at 38px):**
Draw each curve above center, then draw reflected below center at slightly lower opacity (0.5-0.7).

**Smooth curve drawing:**
Use `quadraticCurveTo` to interpolate smoothly through 16 control points (code from RESEARCH.md Pattern: Bezier Curve Through Control Points).

**Bloom passes (locked decision — 3 passes with additive blending):**
After drawing core lines (thin, `lineWidth: 1.5`, `globalAlpha: 0.9`, `shadowBlur: 0`):
1. Bloom pass 1: `globalCompositeOperation: "lighter"`, `globalAlpha: 0.6`, `shadowBlur: 8`, `shadowColor: "#06b6d4"`, `lineWidth: 2`
2. Bloom pass 2: `globalAlpha: 0.3`, `shadowBlur: 16`, `lineWidth: 3`
3. Bloom pass 3: `globalAlpha: 0.15`, `shadowBlur: 24`, `lineWidth: 5`

**State reset after bloom (Pitfall 3):**
```typescript
ctx.globalCompositeOperation = "source-over";
ctx.globalAlpha = 1;
ctx.shadowBlur = 0;
```

**Idle behavior (dim glow at idle — locked decision):**
When `lv < 0.02` (near silence), draw gentle sine wave at 5% max height with a single bloom pass. Use slow sine wave (`sin(t * 1.2)`) for gentle breathing motion. This signals active component without distraction.

**DO NOT** use `getContext("2d", { alpha: false })` — canvas must be transparent (Pitfall 4).
